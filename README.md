Medical RAG System – Intelligent Healthcare Q&A

This project presents an end-to-end Retrieval-Augmented Generation (RAG) system designed to answer medical questions using structured healthcare data downloaded from the HSE (Health Service Executive) website. The system addresses a common limitation of large language models, which is their tendency to generate inaccurate or hallucinated responses when domain-specific knowledge is missing. By integrating a retrieval mechanism with a generative model, this solution ensures that answers are grounded in relevant medical information sourced from publicly available healthcare content.

The architecture of the system follows a structured pipeline where a user query is first processed through a vector database to retrieve the most relevant documents. These documents are created in markdown format from healthcare content collected from HSE pages and organized by disease and topic. The retrieved context is then passed to a language model, such as Llama 3, Qwen, or OpenAI models, which generates a final response based on the provided evidence. This approach improves answer accuracy and reduces hallucination by ensuring that responses are based on retrieved source content rather than model memory alone.

A key strength of this project lies in its data preparation and structuring process. The HSE content was cleaned, organized, and converted into disease-specific markdown files with sections such as overview, symptoms, diagnosis, and treatment. This made the dataset both human-readable and optimized for retrieval tasks. Metadata such as source links and section names was preserved to improve traceability and retrieval quality.

The retrieval system is built using semantic search powered by Sentence Transformers, specifically the MiniLM model. It retrieves the most relevant documents using similarity search and applies additional filtering to improve precision. The system also supports optional reranking techniques to further refine results, ensuring that the best possible context is passed to the language model for final answer generation.

The project supports multiple language models, allowing flexibility in experimentation and evaluation. Models such as Llama 3 and Qwen are used through Ollama for local inference, while OpenAI models are optionally used for evaluation purposes. This approach supports both cost-efficient local development and strong benchmarking when required.

To assess system performance, an evaluation pipeline inspired by RAGAS metrics has been implemented. The evaluation focuses on key aspects such as faithfulness, relevance, and correctness of the generated answers. BLEU score is also used to compare generated responses with expected answers. The evaluation workflow is designed to be lightweight so that experimentation remains practical and cost-conscious.

An interactive user interface has also been developed using Streamlit to demonstrate the system. The interface allows users to select diseases from a dropdown menu, view suggested questions, and receive context-aware answers generated by the RAG pipeline. This provides a practical demonstration of how retrieval and generation work together in a healthcare question-answering setting.

The project is implemented using Python and leverages tools and frameworks such as LangChain, ChromaDB, Sentence Transformers, and Streamlit. It demonstrates skills in natural language processing, vector databases, data preparation, retrieval engineering, and machine learning integration. The modular structure of the codebase includes separate components for ingestion, retrieval, generation, and evaluation, making the system scalable and maintainable.

Future improvements for this project include hybrid search using both keyword and semantic retrieval, stronger reranking models such as cross-encoders, metadata-based filtering for more precise retrieval, and deployment through Docker or cloud platforms. Expanding the dataset and introducing more robust evaluation coverage could further strengthen the system.

This project demonstrates the practical application of combining data engineering, NLP, and machine learning to build a more reliable AI system. By using healthcare information downloaded from the HSE website as the knowledge base, the project shows how domain-specific retrieval can improve the trustworthiness and usefulness of language model responses.
